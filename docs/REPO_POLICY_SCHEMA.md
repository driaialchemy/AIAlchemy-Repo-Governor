# Repo Policy Schema

This document describes the structure of a `CLAUDE.md` policy file as generated by `repo-governor policy-init`.

## File Location

`CLAUDE.md` lives at the root of the repository it governs.

## Schema (Markdown Sections)

### `# CLAUDE.md — Agent Policy`

Required header. Identifies the file as an agent policy document.

---

### `## Project Overview`

| Field | Description |
|-------|-------------|
| `Repository` | Name of the repository directory |
| `Risk Level` | Classification from Repo Governor: LOW, MEDIUM, or HIGH |

---

### `## Allowed Operations`

Bulleted list of operations the agent is explicitly permitted to perform. Defaults generated by `policy-init`:

- Read any source file.
- Run linters, formatters, and the test suite.
- Create new files and directories.
- Edit existing source files.

Customize this list for the specific project.

---

### `## Forbidden Operations`

Bulleted list of hard prohibitions. Defaults:

- Do not delete files without explicit user confirmation.
- Do not commit or push changes automatically.
- Do not read, print, or expose the contents of `.env`, credential, or secret files.
- Do not install packages globally or modify system configuration.
- Do not run destructive commands.

---

### `## Safe Directories`

Specifies which directories the agent may access. The default is all directories except `.git/` and any directory named `secrets/`, `credentials/`, or `private/`.

Add project-specific exclusions here.

---

### `## Environment`

| Field | Description |
|-------|-------------|
| `Language(s)` | Detected programming languages |
| `Build tool` | `pyproject.toml`, `requirements.txt`, or "Not detected" |

---

### `## Notes`

Free-form warnings generated by Repo Governor, such as:

- Detected `.env` files that need to be secured.
- Missing `.gitignore`.

Append project-specific notes here after generation.

## Customization

`policy-init` generates a safe starting point. You should:

1. Review each section and adjust to your project's actual constraints.
2. Commit `CLAUDE.md` to the repository so it persists across agent sessions.
3. Update it when the project's structure or risk profile changes.
